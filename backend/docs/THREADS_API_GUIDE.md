# 📋 Threads API Guide

> **Base URL:** `http://localhost:5000/api/threads`
> **Auth:** Routes marked with 🔒 require a valid `accessToken` cookie (via the `protect` middleware).

---

## Data Model

| Field            | Type     | Description                                       |
| ---------------- | -------- | ------------------------------------------------- |
| `title`        | String   | Thread title*(required)*                        |
| `content`      | String   | Thread body text*(required)*                    |
| `author`       | ObjectId | Auto-set from logged-in user                      |
| `subForum`     | ObjectId | The SubForum this thread belongs to*(required)* |
| `forum`        | ObjectId | Auto-set from `subForum.forum`                  |
| `tags`         | [String] | Lowercase, trimmed tag strings                    |
| `attachments`  | [String] | Cloudinary URLs or Base64 data URIs               |
| `commentCount` | Number   | Defaults to `0` (managed by Comment hooks)      |
| `isPinned`     | Boolean  | Defaults to `false` (Admin-only toggle)         |
| `createdAt`    | Date     | Auto-generated by Mongoose                        |
| `updatedAt`    | Date     | Auto-generated by Mongoose                        |

## Visibility & Lifecycle Matrix

The visibility of threads in **Lists** (Feeds) depends on the status of their parent **Forum**.

| Scenario | Forum Status         | Student/Alumni Feed | Admin Feed (Audit) | Direct URL |
| :------- | :------------------- | :------------------ | :----------------- | :--------- |
| **1**    | Approved & Active    | ✅ **Visible**       | ✅ **Visible**      | ✅ Allowed  |
| **2**    | **Pending Approval** | ❌ **Hidden**        | ✅ **Visible**      | ✅ Allowed  |
| **3**    | **Soft-Deleted**     | ❌ **Hidden**        | ✅ **Visible**      | ✅ Allowed  |
| **4**    | **Rejected**         | ❌ **Hidden**        | ✅ **Visible**      | ✅ Allowed  |

> 💡 **Admin Audit Mode:** Admins always see all threads in list views to facilitate moderation and approval. Non-admins only see threads from "Scenario 1" forums.

### Database Indexes

| Index                              | Purpose                       |
| ---------------------------------- | ----------------------------- |
| `{ subForum: 1, createdAt: -1 }` | Fast SubForum feed queries    |
| `{ forum: 1, createdAt: -1 }`    | Fast Forum-level feed queries |
| `{ tags: 1 }`                    | Fast tag-based filtering      |

## Endpoints

### 1. Create a Thread 🔒

```
POST /api/threads
```

**Request Body:**

```json
{
  "title": "How I got into Google",
  "content": "Here is my full journey...",
  "subForumId": "64a1b2c3d4e5f6a7b8c9d0e1",
  "tags": ["google", "placement", "interview"],
  "attachments": [
    "https://res.cloudinary.com/demo/image/upload/resume.pdf",
    "data:image/png;base64,iVBORw0KGgoAAAANSUhEUg..."
  ]
}
```

**What happens internally:**

1. Validates that `title`, `content`, and `subForumId` are present.
2. Fetches the SubForum document and **auto-sets** the `forum` field from `subForum.forum`.
3. **Sanitizes tags** → lowercased, trimmed, whitespace removed.
4. **Intercepts Base64 strings** in attachments → uploads them to Cloudinary → replaces with secure URLs.
5. Creates and returns the thread.

**Response:** `201 Created`

```json
{
  "status": "success",
  "data": {
    "thread": { "...thread object..." }
  }
}
```

---

### 2. Get Threads by SubForum (Tag-Based)

```
GET /api/threads/subforums/:subForumId
```

**Query Params (optional):**

| Param     | Default | Description      |
| --------- | ------- | ---------------- |
| `page`  | `1`   | Page number      |
| `limit` | `20`  | Threads per page |

**What happens internally:**

1. Fetches the SubForum's `tags` array.
2. Finds all threads where **at least one tag overlaps** (`$in` operator).
3. Sorts by `isPinned` (pinned first) → then `createdAt` (newest first).
4. Applies pagination (`skip` + `limit`).
5. Populates author details (`name`, `avatar`, `role`).

**Response:** `200 OK`

```json
{
  "status": "success",
  "results": 15,
  "data": {
    "pagination": {
      "threads": ["...array of threads..."],
      "totalCount": 42,
      "hasMore": true
    }
  }
}
```

> ⚠️ **Discovery Note:** If the parent Forum is not Approved or is Inactive, this endpoint returns `threads: []` for Students/Alumni. Admins will still see the full list (Audit Mode).

---

### 3. Get Threads by Forum (Forum-Level View)

```
GET /api/threads/forums/:forumId
```

**Query Params (optional):**

| Param     | Default | Description      |
| --------- | ------- | ---------------- |
| `page`  | `1`   | Page number      |
| `limit` | `20`  | Threads per page |

**What happens internally:**

1. Finds **all threads** whose `forum` field matches the given Forum ID.
2. This pulls threads from **every SubForum** under that Forum in one query.
3. Same sorting and pagination as above.

**Response:** `200 OK` *(same structure as SubForum endpoint)*

---

### 4. Get Single Thread

```
GET /api/threads/:threadId
```

**What happens internally:**

1. Finds the thread by its `_id`.
2. Populates author details (`name`, `avatar`, `role`).
3. Returns `404` if not found.

**Response:** `200 OK`

```json
{
  "status": "success",
  "data": {
    "thread": { "...full thread object..." }
  }
}
```

> 🔗 **Direct Access:** This endpoint allows viewing a thread by ID even if its parent Forum is hidden from feeds (Scenario 2/3), provided the user has the link.

---

### 5. Update a Thread 🔒

```
PATCH /api/threads/:threadId
```

**Permissions:**

| Role   | Can Edit? | Can Pin/Unpin? |
| ------ | --------- | -------------- |
| Author | ✅        | ❌             |
| Admin  | ❌        | ✅ *(pin only)* |
| Others | ❌        | ❌             |

> ⚠️ **Cannot change `subForum`**. Sending `subForumId` or `subForum` in the body will return a `400` error.

**Request Body (all fields optional):**

```json
{
  "title": "Updated Title",
  "content": "Updated body text",
  "tags": ["updated", "tags"],
  "attachments": [
    "https://res.cloudinary.com/existing/image.jpg"
  ],
  "isPinned": true
}
```

**Attachment Update Logic:**

- **New Base64 strings** in the array → automatically uploaded to Cloudinary.
- **Removed URLs** (present in DB but missing from the request) → automatically **deleted from Cloudinary** to prevent orphan files.

**Response:** `200 OK`

---

### 6. Delete a Thread 🔒

```
DELETE /api/threads/:threadId
```

**Permissions:**

| Role   | Can Delete? |
| ------ | ----------- |
| Author | ✅          |
| Admin  | ✅          |
| Others | ❌          |

**What happens internally:**

1. Validates ownership (Author or Admin).
2. **Cascade deletes** all attached images from Cloudinary.
3. **Hard deletes** the thread record from MongoDB.

**Response:** `200 OK`

```json
{
  "status": "success",
  "message": "Thread deleted successfully",
  "data": { "thread": null }
}
```

## Error Responses

All errors follow a consistent format:

```json
{
  "status": "fail",
  "message": "Human-readable error description"
}
```

| Status Code | When                                                          |
| ----------- | ------------------------------------------------------------- |
| `400`     | Missing required fields, or attempting to change `subForum` |
| `401`     | Not authenticated (no valid token)                            |
| `403`     | Not authorized (not the author or admin)                      |
| `404`     | Thread or SubForum not found                                  |

## Architecture Overview

```
Request → Route → Controller → Service → Model/Cloudinary → Response
```

| Layer        | File                     | Responsibility                   |
| ------------ | ------------------------ | -------------------------------- |
| Routes       | `threadRoutes.js`      | Maps HTTP methods to controllers |
| Controllers  | `threadController.js`  | Validates input, sends responses |
| Services     | `threadService.js`     | Database queries, business logic |
| Cloud Assets | `cloudinaryService.js` | Upload, delete Cloudinary images |
| Model        | `threadModel.js`       | Schema definition, indexes       |
